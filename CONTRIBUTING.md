# Contributing to Laravel Weather MCP

First off, thank you for considering contributing to Laravel Weather MCP! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please create an issue on GitHub. Please include:
- A clear and descriptive title.
- Steps to reproduce the bug.
- Your PHP and Laravel versions.

### 2. Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue and explain:
- What the feature is.
- Why it would be useful.
- How it should work.

### 3. Submitting Pull Requests
If you want to contribute code, follow these steps:

1. **Fork the Repository**
   Click the "Fork" button at the top right of the repository page.

2. **Clone your Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/Wheater_Mcp.git
   cd Wheater_Mcp
   ```

3. **Create a Branch**
   Create a branch for your feature or bug fix:
   ```bash
   git checkout -b feature/amazing-new-feature
   ```

4. **Make your Changes**
   Write your code. Make sure your code follows the Laravel coding standards (PSR-12).

5. **Commit your Changes**
   Write clear, meaningful commit messages:
   ```bash
   git commit -m "Add humidity data to weather tool"
   ```

6. **Push to your Fork**
   ```bash
   git push origin feature/amazing-new-feature
   ```

7. **Submit a Pull Request**
   Go to the original repository on GitHub and click "Compare & pull request". Describe your changes in detail so we can review them quickly.

## Code of Conduct
Please be respectful and constructive when interacting with other contributors. We want to keep this project welcoming for everyone.

Thank you for your contributions!
